/*
    LIBRARY SYSTEM UI/UX DESIGN GUIDE
    ===================================

    DESIGN PHILOSOPHY:
    This library system interface creates an immersive, warm environment that mimics 
    a physical library while maintaining modern usability standards.


    COLOR PALETTE:
    ===============

    PRIMARY COLORS (Warm Wood Tones):
    - Dark Wood (#3A2D20): Main headers and dominant elements
    - Medium Wood (#785128): Secondary buttons and accents
    - Golden (#B8860B): Highlights and hover states - creates that "special book" feeling

    SECONDARY COLORS (Vintage Book Tones):
    - Vintage Red (#800020): Associated with classic book spines
    - Library Green (#225C3C): Associated with nature and quiet reading spaces
    - Book Orange (#CC5500): Book spine color, creates warm visual interest

    NEUTRAL COLORS (Page & Background):
    - Cream White (#F5F3ED): Main background - feels like aged paper
    - Light Parchment (#FAF8F5): Input fields - feels like writing paper
    - Dark Text (#281E14): Text color - high contrast for readability


    TYPOGRAPHY:
    ============
    Font Family: Segoe UI (modern, readable, professional)

    Heading 1 (Page Titles): 20-22pt, Bold, Dark Wood
    Heading 2 (Section Titles): 14-16pt, Bold, Dark Wood
    Body Text (Labels): 10-11pt, Regular, Dark Text
    Input Labels: 11pt, Regular, Dark Text


    LAYOUT PRINCIPLES:
    ===================
    1. Consistent Padding: 20-30px margins for breathing room
    2. Vertical Rhythm: 10px spacing between elements
    3. Card-based Design: Use panels to create visual separation
    4. Header Bar: Dark wood with golden accents on all major forms
    5. Proper Alignment: Left-align text, center buttons in action areas


    COMPONENT STYLING:
    ==================

    BUTTONS:
    --------
    Primary Buttons (Save, Login, Submit):
    - Background: Medium Wood (#785128)
    - Text: Cream White
    - Border: 1px Dark Wood
    - Hover: Golden (#B8860B)
    - Cursor: Hand pointer
    - Height: 36-45px for adequate click area

    Secondary Buttons (Edit, Cancel):
    - Background: Cream White
    - Text: Dark Text
    - Border: 1px Soft Border
    - Hover: Light gray

    ACTION BUTTONS (Dashboard shortcuts):
    - Vary background colors using vintage palette
    - Add 2px border for definition
    - Include emoji/icons for visual interest
    - Large size (200x120px) for easy clicking

    TEXT BOXES / INPUT FIELDS:
    --------------------------
    - Background: Light Parchment (#FAF8F5)
    - Text Color: Dark Text (#281E14)
    - Border: 1px subtle border or FixedSingle
    - Padding: 8-12px inside for comfortable typing
    - Font: Segoe UI, 11pt
    - Height: 32px for single-line inputs

    LABELS:
    -------
    - Font: Segoe UI, 10-11pt
    - Color: Dark Text (#281E14)
    - Margin Below: 8px spacing from input field

    DATA GRIDS:
    -----------
    - Header: Medium Wood background, Cream text, Bold font
    - Body: Alternating white and very light cream rows
    - Borders: Soft border color (#C8B4A0)
    - Row Height: 28px minimum for text readability
    - Selection: Use blue highlight (#4682B4)

    GROUP BOXES:
    -----------
    - Title Font: Segoe UI, 10pt, Bold
    - Title Color: Dark Wood
    - Background: Cream White
    - Border: Subtle, uses native GroupBox

    PANELS:
    -------
    - Use dark wood panels as headers/separators
    - Use cream/parchment for content areas
    - Maintain consistent padding


    SPACING & ALIGNMENT:
    =====================
    Form Margins: 20-50px padding around content
    Section Spacing: 20-30px between major sections
    Element Spacing: 10-15px between form fields
    Button Groups: 8-10px between buttons

    LEFT ALIGNMENT:
    - Form labels: Left-aligned at consistent x position
    - Input fields: Aligned below or beside labels

    CENTER ALIGNMENT:
    - Page titles: Centered in header
    - Buttons in dialog boxes: Centered or right-aligned
    - Welcome messages: Left or center depending on context

    RIGHT ALIGNMENT:
    - Action buttons in data grids
    - Close/Exit buttons in dialogs


    RESPONSIVE CONSIDERATIONS:
    ===========================
    - Minimum form width: 600px (Login)
    - Minimum form width: 900px (Data forms)
    - Use DockStyle.Fill for proper resizing
    - Maintain proper padding on resize


    USER EXPERIENCE PRINCIPLES:
    ============================
    1. Visual Hierarchy: Headers > Labels > Input fields
    2. Affordance: Buttons clearly look clickable
    3. Feedback: Hover states and color changes
    4. Consistency: Same element types look and behave identically
    5. Accessibility: High contrast for readability
    6. Efficiency: Quick access buttons on homepage


    IMPLEMENTATION GUIDELINES:
    ==========================

    For All Forms:
    1. Call UIHelper.StyleForm() in form constructor
    2. Create header panel using UIHelper.CreateHeaderPanel()
    3. Use proper color scheme values
    4. Apply consistent spacing and padding

    For Input Forms:
    1. Group related fields
    2. Use consistent label positioning
    3. Style all buttons using UIHelper.StyleButton()
    4. Add separator lines between sections

    For List/Grid Forms:
    1. Style DataGridView using UIHelper.StyleDataGridView()
    2. Add search bar at top
    3. Add action buttons (Edit, Delete, Add)
    4. Implement proper column widths


    FEEL & ATMOSPHERE:
    ===================
    This design should evoke:
    ? Warmth and comfort (wooden tones)
    ? Knowledge and tradition (vintage colors)
    ? Quiet concentration (calm color palette)
    ? Accessibility (clear, readable typography)
    ? Professionalism (organized layout)
    ? Trust (consistent, predictable interface)

    Users should feel like they're using a digital extension of a physical library,
    not a generic database application.


    REFERENCE FORMS:
    =================
    - Login Form: Header + Content pattern with centered form
    - Homepage: Header + Menu + Content with shortcut buttons
    - Books/Borrower: Header + Sidebar + GridView pattern
    - Reports: Header + Filter panel + Full-width display
*/
